## text

### Source (en)

~~~text
If you need to perform many similar calculations of different input data (e.g. samples or simulations), these can often be executed at the same time on different nodes in a compute system. This is sometimes called "trivial parallelism". Even though it is named "trivial", this still requires that the work flow and workflow system support this (e.g. by making sure all file names are unique, and that not many threads are trying to read or write the same file at the same time). 

~~~

### Translation (zh_Hant)

~~~text
如果你需要對不同的輸入資料（例如樣本或模擬資料）執行許多相似的計算，這些計算通常可以在計算系統的不同節點上同時執行。這有時稱為「瑣碎平行化（trivial parallelism）」。雖然名稱中有「瑣碎」一詞，但這仍然要求工作流程與工作流程系統能夠支援此種做法（例如確保所有檔名皆為唯一，且不會有過多執行緒在同一時間嘗試讀取或寫入同一個檔案）。 

~~~
